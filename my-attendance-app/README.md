IoT NFC Attendance System – Web Dashboard

*Project Description:*

The Web Dashboard is the application layer of the IoT NFC Attendance System.
It provides professors with an interface to manage attendance sessions, organize classes, and monitor student presence in real time using NFC identification technology.

The platform communicates with the cloud backend through Supabase services and receives attendance events generated by the NFC hardware system. The objective of this web application is to replace manual attendance processes with an automated, reliable, and scalable solution.

*Live Demonstration:*

Website URL
https://iot-nfc-attendance.netlify.app/

*System Objectives:*

1. Automate student attendance registration
2. Reduce attendance fraud and proxy presence
3. Provide real-time monitoring of classroom activity
4. Simplify academic session management
5. Integrate IoT hardware with a modern web interface

*Core Features:*

1. Secure authentication system for professors
2. Creation and management of academic classes
3. Attendance session activation and termination
4. Real-time visualization of student attendance
5. Automatic attendance recording based on NFC scans
6. Cloud synchronization using Supabase database services
7. Responsive and modern user interface

*Technology Stack:*

*Local Development Setup:*

1. Clone the repository:

git clone https://github.com/abdelhakhaouari/iot-nfc-attendance-system.git

2. Navigate to the web application directory:

cd my-attendance-app

3. Install project dependencies:

npm install

4. Run the development server:

npm run dev

5. Environment Configuration:

Create a file named .env in the project root directory and define the following variables:

VITE_SUPABASE_URL=YOUR_SUPABASE_URL
VITE_SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY

*Application Workflow:*

1. The professor authenticates through the login interface.
2. A class is selected or created.
3. An attendance session is activated.
4. NFC hardware sends student UID data to Supabase.
5. The dashboard updates attendance records instantly.

*System Architecture Role:*

Within the overall IoT system, this website represents the application layer responsible for visualization, interaction, and attendance management.

NFC Reader → Controller → Supabase Backend → Web Dashboard
